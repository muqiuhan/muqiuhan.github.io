编程语言中的环境，名称以及状态语义

Muqiu Han
2022-06-26
Semantics

这篇文章直观的介绍了编程语言中名称，环境，内存的概念，以及对这些概念的正式描述，它使我们能够熟悉很多通用的编程语义方法。

Introduction

许多高级编程语言都使用名称来表示程序处理的实体, 这些名称通常称为标识符，请注意它们是根据所选语言的词法规则构造的，它们可用于表示程序特定的值或在执行期间计算的值。它们也可以表示位置（即内存中的地址），这种表示位置的名称被称为可变变量。根据语言中存在的结构，标识符还可以表示运算符、函数、过程、模块、对象等。例如，pi 常用于表示 π\piπ 的近似值； + 也是一个标识符，表示一个加法运算符，通常放在两个操作数之间，即在中缀位置，如 2 + 3 。表达式 2 * x + 1 使用标识符 x 并计算它的值，我们需要知道 x 表示的值。查找与给予标识符关联的值是所有高级语言的核心机制。语言的语义提供了这种机制的模型。

注意: 下面给出的所有语言、指令、算法等的正式定义都是用OCaml描述的.

Environment, memory and state

Evaluation environment

令 XXX 为一组标识符，而 VVV 为一组值。标识符 x ∈ XXX 与值 v ∈ VVV 的关联称为绑定（标识符与其值的绑定）。绑定的集合 称为 Env，Env是执行环境或求值(Evaluation)环境。 Env(x)表示Env中的标识符x关联的值。环境集记为 EEE。

在实践中，实际使用的标识符 XXX 的集合是有限的：通常，我们只考虑那些出现在程序中的标识符。因此，环境可以由绑定列表表示，也就是Env，其定义如下:

Memory

下面介绍内存模型类似于正常的物理内存，内存的状态是通过将一个值与存储它的单元的位置相关联来描述的。位置本身也被视为值，称为引用。高级语言允许我们通过绑定到 c 的引用 r 的标识符 x 来命名包含值 v 的位置 c：

令 RRR 表示为引用的集合，VVV 为值的集合。引用r ∈ RRR 值 v ∈ VVV , r和v的关联由一组(r, v) 表示，这些(r, v)的集合为 Mem，也就是内存。 Mem® 表示存储在 Mem 中的引用 r 处的值v。设 MMM 为内存集。在实践中，实际使用的引用集是有限的：所以我们通常只考虑程序使用的那些位置。这意味着内存也可以用一个列表来表示，也称为 Mem，它的定义如下:

[(r1,Mem(r1)),(r2,Mem(r2)),⋯ ,(rn,Mem(rn))][(r_1,Mem(r_1)), (r_2, Mem(r_2)), \cdots , (r_n, Mem(r_n))] [(r1​,Mem(r1​)),(r2​,Mem(r2​)),⋯,(rn​,Mem(rn​))]

也就是说，内存中存在一对 (r, v) 记录了在该位置执行了初始化或写入操作。每个被引用的存储单元都可以通过读取来查询，并且可以通过写入来分配一个新的值。在这种情况下，先前保存在单元格中的值会被删除，它已经“失效”。也就是说，引用必须是唯一的。编程语言中的标识符当然可以重复，但是计算机中不能出现两个相同的内存地址。

向位置（地址） r 处写入值 v 则会将内存 Mem 转换为表示为 Mem[r := v] 的内存；如果一个值已经存储在 Mem 中的 r 位置，那么这个旧值会被新的值 v 替换；否则，就向 Mem 添加一个新的(r, v)，这是一个写入操作。与环境的情况相反，没有掩蔽。在已包含值的位置 r 处写入新值 v 会删除旧值 Mem®，所以这个操作可以表示为:

[(r1,v1),⋯ ,(ri,vi),⋯ ,(rn,vn)][ri:=vi′]=[(r1,v1),⋯ ,(ri,vi′),⋯ ,(rn,vn)][(r_1, v_1), \cdots, (r_i, v_i), \cdots, (r_n, v_n)][r_i := v^\prime_i] \\ = [(r_1, v_1), \cdots, (r_i, v^\prime_i), \cdots, (r_n, v_n)] [(r1​,v1​),⋯,(ri​,vi​),⋯,(rn​,vn​)][ri​:=vi′​]=[(r1​,v1​),⋯,(ri​,vi′​),⋯,(rn​,vn​)]

而内存域表示为dom(Mem)，dom(Mem取决于当前环境，它表示引用空间，可以从当前执行环境中的绑定和非屏蔽标识符（直接或间接）访问这些引用空间。将绑定 (x, r) 添加到环境 Env 具有双重效果: 创建 (x,r)⊕Env(x, r) \oplus Env(x,r)⊕Env 的同时会将 Mem 扩展到 Mem[r := v]。

注意：取决于所讨论的语言或程序，值 v 可能仅在引入 x 时提供，或者稍后提供，或者从不提供。如果在第一次使用之前没有提供值，则程序的结果是不可预测的，这会导致被称为初始化错误的错误。事实上，一个位置r总是包含一个值v，如果它没有被程序明确确定，那它就不应该出现在当前的计算环境中。

注意: 在上面的介绍中，赋值只涉及可变变量。同样，去引用运算符!只能应用于引用以获取此位置的存储值。因此，从给定引用将值分配给可变变量需要使用专用的语法结构：x:=!x+1x := !x + 1x:=!x+1 。然而，在许多语言中，赋值并没有明确提到解引用运算符，并且变量的语法使用在赋值的两边是相同的： x=x+1x = x + 1x=x+1 。因此，标识符 x 表示两个不同的概念：

  1. 赋值符号左侧的值 Env(x)
  2. 和赋值符号右侧的值 Mem(Env(x))

这些语言根据变量在赋值中的位置来掩盖变量的不同角色:

  • 当变量 x 位于赋值的左侧时，它被称为左值(l-value)，它表示应存储要分配的值的位置。这样，它就充当了一个指针。
  • 当变量 x 位于赋值的右侧时，它被成为右值(r-value)，并隐含了取消引用的指针——因为要获取的值位于变量表示的位置。

因此，在表达式 x=x+1x = x + 1x=x+1 中，即使从句法角度以相同的方式使用 x，右侧的实例也隐含表示 !X。在某些语言中，变量声明隐含地涉及创建引用，除非另有说明；变量的名称表示编译器将存储分配给它的值的位置。
